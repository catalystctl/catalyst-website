Welcome! This guide gets you from zero to a running Catalyst panel in about five minutes. No prior experience needed — just Docker.

> 💡 **Already have Docker?** Skip to [Step 1](#step-1-run-the-one-line-installer).

---

## Prerequisites

| Requirement | Minimum | Notes |
|---|---|---|
| **Docker** 20.10+ **or** **Podman** 4.0+ | Required | Docker Compose v2 plugin or `podman-compose` |
| **Ports** | 8080, 3000, 2022 | Adjust in `.env` if conflicts |
| **RAM** | 2 GB | 4+ GB recommended |
| **Disk** | 10 GB | SSD recommended |

> **Don't have Docker?** Install it first: [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Mac/Windows) or `sudo apt install docker.io docker-compose-v2` (Ubuntu/Debian).

---

## Step 1: Run the One-Line Installer

Open a terminal and run:

```bash
curl -fsSL https://raw.githubusercontent.com/catalystctl/catalyst/main/install.sh | bash
```

This downloads the `catalyst-docker/` folder with pre-built images and generates a `.env` file for you.

> 🔒 The script auto-generates strong secrets for you — no need to run `openssl` yourself.

---

## Step 2: Configure

```bash
cd catalyst-docker
nano .env
```

Set these three values:

| Variable | What to enter | Example |
|---|---|---|
| `PUBLIC_URL` | The URL you'll open in your browser | `http://localhost:8080` or `http://192.168.1.78:8080` |
| `POSTGRES_PASSWORD` | A strong database password | `MySuperSecretDbPass123!` |
| `BETTER_AUTH_SECRET` | Session encryption key (already generated by the script) | Leave as-is, or regenerate with `openssl rand -base64 32` |

> **Tip:** Find your LAN IP with `hostname -I | awk '{print $1}'` if you want to access Catalyst from another device on your network.

> **Podman users:** Make sure `FRONTEND_PORT=0.0.0.0:8080` is set (this is the default). Rootless Podman can't bind ports below 1024.

---

## Step 3: Start

```bash
docker compose up -d
```

That's it! The command downloads images and starts everything in the background.

---

## Step 4: Verify

```bash
docker compose ps
```

You should see four containers, all healthy:

| Container | Status | Port |
|---|---|---|
| `catalyst-postgres` | healthy | `127.0.0.1:5432` |
| `catalyst-redis` | healthy | `127.0.0.1:6379` |
| `catalyst-backend` | healthy | `127.0.0.1:3000`, `0.0.0.0:2022` |
| `catalyst-frontend` | running | `0.0.0.0:8080` |

> If any container shows `unhealthy` or `restarting`, check logs: `docker compose logs -f <service>`.

---

## Step 5: Login

1. Open your `PUBLIC_URL` in a browser (e.g., `http://localhost:8080`).
2. Click **Create Account** and register your first user.
3. 🎉 **You are now the administrator!**

> **No seeding required.** The first user to register automatically becomes the admin.

## 🩹 Common Issues

<details>
<summary><strong>Port already in use</strong></summary>

If you see `bind: address already in use`, another service is using a Catalyst port. Edit `.env` and change the conflicting port:

```bash
# Example: if 8080 is taken
FRONTEND_PORT=0.0.0.0:8081
PUBLIC_URL=http://localhost:8081
```

Then restart: `docker compose up -d`.
</details>

<details>
<summary><strong>Podman: <code>rootlessport cannot expose privileged port 80</code></strong></summary>

Rootless Podman can't bind ports below 1024. Use the default high port (8080) or allow privileged ports:

```bash
echo 'net.ipv4.ip_unprivileged_port_start=80' | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```
</details>

<details>
<summary><strong>Can't connect to database / backend keeps restarting</strong></summary>

1. Check PostgreSQL is running: `docker compose ps postgres`
2. Check logs: `docker compose logs postgres`
3. Make sure `POSTGRES_PASSWORD` is set in `.env`
4. If you previously started with a different password, reset everything:
   ```bash
   docker compose down -v
   docker compose up -d
   ```
   ⚠️ This deletes all data — only use on a fresh install.
</details>

<details>
<summary><strong>Panel shows a blank page</strong></summary>

Make sure `PUBLIC_URL` exactly matches the URL in your browser. Mismatches cause CORS errors. If you accessed via `http://192.168.1.78:8080`, set `PUBLIC_URL=http://192.168.1.78:8080` (not `localhost`).
</details>
